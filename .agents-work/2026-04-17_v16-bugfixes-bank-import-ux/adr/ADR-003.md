# ADR-003: Chart Library — react-native-gifted-charts

**Session:** 2026-04-17_v16-bugfixes-bank-import-ux  
**Date:** 2026-04-17  
**Status:** Accepted  
**AC:** AC-614

---

## Context

V1.6 adds a Reports screen with two chart types:
- **Bar chart**: last 6 months of spending per category.
- **Pie chart**: current-month category spending distribution.

Requirements:
- Expo SDK 52 / React Native 0.76.5 compatible.
- MIT license.
- No native modules beyond what Expo managed workflow supports.
- Reasonable bundle size for the Android-first pilot.
- Zero cost.

Two candidates are specified in the constraints: `victory-native` and `react-native-gifted-charts`.

---

## Decision

Use **`react-native-gifted-charts`** (MIT, ~300 KB gzipped) for both bar and pie charts.

---

## Alternatives Considered

### A. `victory-native` (v40+, uses Skia)
- **Pros**: Polished, well-documented, actively maintained by FormidableLabs; uses `@shopify/react-native-skia` for hardware-accelerated rendering.
- **Cons**: Requires `react-native-reanimated` ≥ 3 (already installed) and `@shopify/react-native-skia`, which needs a native build step and is not included in Expo Go. In bare/development build workflow it works, but adds ~4 MB to the Android APK just for Skia. Overkill for a pilot with ≤ 10 users.
- **Rejected**: Skia dependency adds significant build complexity and APK size without a meaningful UX benefit at pilot scale.

### B. `react-native-gifted-charts`
- **Pros**: Pure JS (uses `react-native-svg` via Expo's SVG package — already commonly installed); bar, line, and pie charts in one package; simple declarative API; MIT licensed; no Skia or heavy native modules.
- **Pros**: `react-native-svg` is the lighter, more Expo-compatible SVG runtime (~1 MB).
- **Cons**: Less visually premium than Skia-based rendering; limited animation options.
- **Accepted**: Sufficient quality for a pilot, minimal native surface, integrates cleanly with Expo managed workflow.

### C. Custom SVG charts using `react-native-svg` directly
- **Pros**: Zero additional dependency; full control.
- **Cons**: Bar + pie charts are non-trivial to implement from scratch; distracts from product delivery.
- **Rejected**: Engineering cost not justified when gifted-charts is MIT and sufficient.

---

## Consequences

- **Positive**: Low native footprint; compatible with Expo managed build.
- **Positive**: Single package covers all chart types needed for V1.6 (bar, pie, line for future trends).
- **Negative**: Less animation flexibility than Skia-based alternatives.
- **Migration path**: If V2 requires more advanced charts (animations, interactions, custom rendering), switching to `victory-native` is scoped to the `ReportsScreen` component — the data shape is chart-library-agnostic.

---

## Notes

- Peer dependency: `react-native-svg` — install via `npx expo install react-native-svg`.
- Install command: `npx expo install react-native-gifted-charts react-native-svg`.
- The `ReportsScreen` component receives plain data arrays `{ label: string, value: number }[]` and maps them to chart props; chart library details are encapsulated in `ReportsScreen` and not exposed to the data layer.
